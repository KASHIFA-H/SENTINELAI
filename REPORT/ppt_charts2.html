How the ML Model Works

SentinelShield uses a Weighted Linear Combination Model — a supervised scoring approach where each detection signal contributes a weighted value to the final attack probability score.

P(attack) = Σ wᵢ × sᵢ

w₁ = 0.25 → Behavioral anomaly
w₂ = 0.30 → Entropy spike
w₃ = 0.20 → Exfiltration risk
w₄ = 0.15 → Canary risk
w₅ = 0.10 → Network anomaly

Threshold: P ≥ 0.70 → HIGH RISK

The ransomware family classifier uses pattern matching on the attack signature (XOR key pattern + rename behavior + modification rate) to identify LockBit 3.0, BlackCat/ALPHV, or Cl0p.

Why this approach?
Entropy is weighted highest (0.30) because it is the most reliable indicator of encryption — near-zero false positives above 7.0 bits. Behavioral is second (0.25) as rapid modification is the earliest detectable signal. Canary and honeypot are binary triggers — when hit, they dominate the score regardless of weights.

Formula: Attack Probability Score
P = 0.30×E + 0.25×B + 0.20×X + 0.15×C + 0.10×N
E=Entropy · B=Behavioral · X=Exfil · C=Canary · N=Network
Why Weighted Linear Combination?
✓ Interpretable
Each weight has a clear meaning — judges and operators can understand why the score is high
✓ Real-time capable
O(1) computation — no model inference latency, score updates in milliseconds
✓ No training data needed
Works from day 1 — unlike neural networks that need thousands of attack samples
Future: RL Agent Optimization
Replace static weights with a Reinforcement Learning agent that learns optimal weights for each specific OT environment — eliminating false positives from legitimate batch processes
