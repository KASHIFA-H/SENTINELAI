Data Collection

SentinelShield collects real-time telemetry from the file system using Python's watchdog library. Every file event is captured and fed into the feature extraction pipeline.

File System Events

CREATE · MODIFY · DELETE · RENAME
Captured every 3 seconds via watchdog observer. Each event records: filepath, timestamp, event type, file size.

File Content Sampling

Up to 64KB of each modified file is read in binary mode for entropy calculation. Raw bytes (0–255) are sampled to compute byte frequency distribution.

Process Metadata

psutil monitors running processes. Process name, PID, command line, and file access patterns are tracked to identify suspicious processes.

Feature Extraction

Five features are extracted from raw file system events. Each feature is normalised to a 0–100 scale before being fed into the scoring model.

Feature 1: Shannon Entropy (H)
H = -Σ p(x) × log₂(p(x))
where p(x) = count(byte x) / total_bytes
Range: 0.0 – 8.0 bits/byte
Normalised: (H / 8.0) × 100

Reads up to 64KB of file content. Counts frequency of each byte value (0–255). High entropy (>7.0) = likely encrypted.

Feature 2: Modification Rate
rate = count(modifications in 5s window)
Threshold: ≥ 3 modifications = suspicious
Normalised: min(rate × 10, 100)

Sliding 5-second window tracks how many times each file is written. Ransomware modifies files 4–10× per second.

Feature 3: Rename Pattern
pattern_score = 50 if .locked extension
pattern_score = 30 if unknown extension
Binary feature: 0 or 50

Detects file rename events. Adding .locked, .encrypted, .crypt extensions is a strong ransomware indicator.

Feature 4: Canary File Access
canary_score = 100 if canary touched
canary_score = 0 otherwise
Binary feature: 0 or 100

Hidden sentinel file. Any access = confirmed ransomware. Near-zero false positive rate. Adds 100 points instantly.

Feature 5: Honeypot Access
honeypot_score = 100 if honeypot read
bulk_read_score = 35 if 5+ reads/10s
Binary + rate feature

Four fake credential files. Any read = exfiltration attempt. Bulk read anomaly detects silent data copying.

ML Model: Weighted Linear Scoring

SentinelShield uses a Weighted Linear Combination Model — a deterministic scoring approach that requires no training data and works immediately on any environment.

P(attack) = Σ (wᵢ × sᵢ) / 100

s₁ = min(threatScore/2, 50) [Behavioral]
s₂ = 90 if critical else 15 [Entropy]
s₃ = 85 if honeypot else 10 [Exfil]
s₄ = 95 if critical else 5 [Canary]
s₅ = random(0,20) + offset [Network]

w = [0.25, 0.30, 0.20, 0.15, 0.10]

Threshold: P ≥ 0.70 → HIGH RISK
Why Weighted Linear vs Neural Network?

Neural networks require labelled training datasets (thousands of attack samples). In OT environments, labelled ransomware data is scarce and environment-specific. The weighted linear model works immediately with zero training, has 100% interpretability (you can explain every decision), and runs in microseconds on any hardware.

Ransomware Family Classifier

A pattern-matching classifier identifies the most likely ransomware family based on the attack signature observed. Activated only when threat score exceeds 30 or status is critical.

Classification Logic
IF xor_encryption AND .locked_rename:
  LockBit 3.0 → 72% confidence

IF entropy_spike_profile AND rapid_mod:
  BlackCat/ALPHV → 18% confidence

IF file_modification_rate_pattern:
  Cl0p → 10% confidence

Confidence sums to 100% (softmax-like)
Why LockBit 3.0 at 72%?

The simulator uses XOR encryption with key 0xAB and renames files with .locked extension. This exact pattern — XOR + .locked — is the signature of LockBit 3.0, the most active ransomware family globally in 2025 (Dragos Year in Review). The 72% confidence reflects the statistical prevalence of this pattern in real-world OT attacks.
